# Введение
## Что это такое и зачем
Это программируемый процессор построенный на комбинаторах, без модов. Он способен выполнять программы написанные на собственном наборе команд.

Процессор состоит из аналогов реальной комьютерной архитектуры: Регистров, АЛУ, ОЗУ, ПЗУ и автомата состояний.

Прикладное применение в игре у него простое: использовать там, где есть необходимость в хранении информации, ветвистой логике и циклах.

## Краткие характеристики
Большую часть времени выполнения программ занимает переключение между командами процессора. Это время зависит от выбранного конвеера под тактовый генератор. Отдельные блоки автомата состояний сами по себе выполняются быстро (от 2 до 10 тактов - зависит от команды).

# Архитектура
## Ключевые элементы
1) Красная шина - он же красный провод, используется только для команд (в рамках общения между элементами, внутри блоков правило может нарушаться)
2) Зеленая шина - он же зеленый провод, используется только для данных (в рамках общения между элементами, внутри блоков правило может нарушаться) 
3) Постоянная память (далее __ПЗУ__) - набор константных комбинаторов, могут работать только в режиме чтения. Адресация начинается с `1`.
4) Оперативная память (далее __ОЗУ__) - ячейки памяти с адресацией которые поддерживают чтение\запись\очистку. Адресация начинается с `1`.
5) __Регистры__ - ячейки памяти, похожие на ОЗУ, которые используются напрямую в некоторых элементах, например в __АЛУ__. Адреса начинаются с `1`, всего 8 регистров.
6) __АЛУ__ - набор почти всех доступных в Factorio операций над сигналами, напрямую подключен к первому и второму регистру.
7) Автомат состояний (далее __Автомат__) - переводит высокоуровневые команды в низкоуровневые и определяет в каком тике какая низкоуровневая команда будет выполнена. Напрямую подключен к 7му и 8му регистрам. По сути это набор комбинаторов с примерно следующим кодом `IF __опкод__ == [L]=[1] AND [T] == 1 -> [A]=[0]`
8) Тактовый генератор (далее __Генератор__) - считает тики для __Автомата__ и переключает индекс текущего выполняемого опкода из __Листинга__.
9) Листинг программы (далее __Листинг__) - набор константных комбинаторов, состоящих из описания команд и аргументов. Адресация начинается с `1`.
## Схема взаимодействия блоков
1) Для старта системы необходимо на шину команд подать __сигнал__ `[I]=1`. Это отпускает тактовый конвеер, происходит переключение индекса __опкода__ на первую команду.
2) __Листинг__ реагирует на изменение индекса и выдаёт на шину новый __опкод__ для __Автомата__
3) __Автомат__ разделен на блоки, каждый из которых обрабатывает свой __опкод__. Блок сравнивает входящих __опкод__ и отправляет в __Генератор__ сигнал `[T]=1`, что запускает генерацию тиков.
4) __Автомат__ реагирует на изменение тиков и отправляет __сигнал__ в систему для других элементов системы (__ОЗУ__, __ПЗУ__, __Регистры__ и так далее). Правильный порядок отправляемых __сигналов__ делает возможность совершать операции над данными.
5) Последний __сигнал__ выходящий из __Автомата__ это `[I]=1`, что вызывает переключение индекса __опкода__. Далее переходим к пункту 2 и всё по новой.
6) Сброс индекса __опкода__ происходит при получении __сигнала__ `[P]=-1` (Перезапуск программы).
```
Генератор -> Листинг -> Автомат -> Шина -> ОЗУ\ПЗУ\Регистры и т.д.
    ↑                   ↓    ↑       |
    |                  Генератор     ↓
    |_______________________________[I]=1

```
## Красная и зеленая шина
Если вы будете добавлять свои элементы системы, то запомните.
Красный провод - шина команд. Только управляющие сигналы, адреса и флаги. Никогда не выводите сюда данные.
Зеленый провод - шина данных. На самом деле данными могут быть любые сигналы - тут ограничений нет.

Такое разделение понадобилось по одной причине: в Factorio сигналы на одном проводе складываются.
Чтобы не ограничивать себя в пуле сигналов для данных - в явном виде был выделен отдельный провод под команды.
## Регистры
1) `R1` - первый аргумент для __АЛУ__ операций.
2) `R2` - второй аргумент для __АЛУ__ операций.
3) `R3` - результат выполнения __АЛУ__ операций.
4) `R4` - свободный
5) `R5` - свободный
6) `R6` - свободный
7) `R7` - указатель на адрес вершины стека.
8) `R8` - хранит флаг `[#]=0` или `[#]=1` для условных переходов. Записать значение можно через `LOAD` или `MOVE`.

# Система команд
## Соглашения
Символ внутри квадратных скобок обозначает отправляемый сигнал, а цифра после знака равно - значение сигнала. Например запись `[F]=1, [0]=5, [1]=3` обозначает, что нужно отправить 3 сигналa: сигнал F равный 1, сигнал 0 равный 5 и сигнал 1 равный 3м.
## Уровни команд
Система содержит 2 уровня команд: верхнего и нижнего уровня.
1) Команда верхнего уровня (__опкод__). Это то что пишет программист в листинге. Каждый опкод описывает одно логическое действие:
```
[L]=1, [0]=5, [1]=1    // загрузить из ОЗУ[5] в R1
[+]=1                  // сложить R1 + R2 → R3
[J]=1, [0]=10          // прыгнуть на строку 10
```
2) Команда нижнего уровня (__сигнал__). Это то что генерирует автомат состояний внутри системы. Один опкод разворачивается в несколько сигналов которые выполняются по очереди в строго определённые такты:
```
[L]=1  →  T=1: [A]=5         // подать адрес на ОЗУ
          T=4: [R]=1, [L]=1  // загрузить данные в регистр
          T=6: [I]=1         // конец операции
```
Нижний уровень важен только при расширении системы новыми __опкодами__.

### Команды АЛУ
Так как в Factorio сигналы очень разнообразны, то было принято решение принять сигнал `[#]` как число.
Например для циклов итератор будет храниться как `[#]=1`, потом `[#]=2` и так далее.
Ниже в режимах работы будет запись `[Каждый]` x `[#]` - она означает произвести АЛУ операцию между каждым сигналом и числом.

Перед этим блоком некоторые сокращения:
- `[АК]` - Сигнал арифметического комбинатора
- `[СК]` - Сигнал сравнивающего комбинатора
- `[ВК]` - Сигнал выбирающего комбинатора
- `[ЛК]` - Сигнал "Любое качество"
- `[ВХОД]` - Сигнал "Вход"
- `[РП]` - Сигнал "Размер Пачки"
- `[ДР]` - Сигнал "Деталь ракеты"


Режимы работы арифметического комбинатора:
1) `[Команда]=1`: `[Каждый]` x `[Каждый]` = `[Каждый]`
2) `[Команда]=2`: `[Каждый]` x `[#]` = `[Каждый]`
3) `[Команда]=3`: `[#]` x `[Каждый]` = `[Каждый]`

Режимы работы сравнивающего комбинатора:
1) `[Команда]=1`: `[Каждый]` x `[Каждый]` = `[Каждый]`
2) `[Команда]=2`: `[Каждый]` x `[#]` = `[Каждый]`
3) `[Команда]=3`: `[#]` x `[Каждый]` = `[Каждый]`

| `[АК]=1, [&]=1` | `R3` = `R1` AND `R2` (Логическое И )|

## Добавление нового опкода
Опкоды обрабатывает Автомат состояний. Для новой команды нужно добавить новый блок. Каждый блок подключен к 4 шинам одновременно и в некоторых случаях ещё и к регистрам.
1) Первая шина - красный провод опкодов от __Листинга__ - делаем простую сверку сигнала и пропускаем внутрь блока в случае совпадения.
2) Вторая шина - источник сигнала для тактового генератора (`[T]=1`), посылается только в том случае, если опкод прошёл внутрь блока автомата. Тем самым мы гарантируем, что тактовый генератор не будет работать в пустую.
3) Третья шина - выход тактового генератора - текущий номер тика. Используется что бы сигналы из автомата выходили только в нужные моменты времени
4) Четвёртая шина - выход блока автомата (Красный и Зеленый провода). Подключен ко входам Регистров\ОЗУ\ПЗУ\АЛУ и всех остальных элементов системы.

Обязательно убедитесь, что опкод уже не используется. А также что ваши кастомные сигналы, для ваших элементов системы не пересекаются с уже существующими.
